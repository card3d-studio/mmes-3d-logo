Morehead Montessori Logo Embed Test Harness

Use this page to test the live logo under realistic container sizes, backgrounds, and layout constraints before handing it to the school. Each scenario embeds the live GitHub Pages demo so you can spot sizing, transparency, accessibility, and fallback issues quickly.

Open live logo demo
What this tests Transparency, sizing, container behavior, mobile fit, dark/light backgrounds, and basic embed resilience.
What this does not fully test Direct DOM-level integration issues inside a school CMS. For that, later we should also make a true embeddable build.
Best workflow Keep this harness in the repo, then add quick notes under each scenario as problems are discovered and fixed.

Reduced-motion check

Turn on Reduce motion in your system accessibility settings, then reload this page. Confirm the fallback behavior matches your intended design.

  • Does the fallback image appear consistently?
  • Is the explanatory text still appropriate?
  • Does the logo remain visually acceptable when interactivity is suppressed?

No WebGL / JS Failure

Manual checks for failure states that still need to feel intentional.

  • Temporarily disable JavaScript and reload the live logo page.
  • Confirm the fallback image appears and the screen-reader text still makes sense.
  • Use a browser/device with WebGL restrictions if available and confirm the static state is acceptable.
  • Check that broken assets do not leave an empty box or confusing message.

This card is documentation-driven because browsers do not let a test page reliably disable another iframe's WebGL or JavaScript in a realistic way.
